# Legally — Architecture & Delivery Docs

This folder contains the core planning and execution documents for the `Legally` AI-native legal platform.

## Documents

- [`backend-architecture-plan.md`](./backend-architecture-plan.md)
  - Backend stack, service architecture, data model, APIs, security, observability, and 12-week backend plan.

- [`frontend-architecture-plan.md`](./frontend-architecture-plan.md)
  - Frontend stack, app structure, UX modules, state strategy, accessibility, testing, and 12-week frontend plan.

- [`end-to-end-delivery-roadmap.md`](./end-to-end-delivery-roadmap.md)
  - Integrated execution roadmap combining backend and frontend milestones, sprint plan, release gates, KPI targets, and pilot readiness.

## Recommended Reading Order

1. `end-to-end-delivery-roadmap.md`
2. `backend-architecture-plan.md`
3. `frontend-architecture-plan.md`

## Usage

- Treat these docs as the source of truth for MVP scope and sequencing.
- Update them at the end of each sprint to reflect architecture decisions and delivery changes.
- Keep all naming, environments, and milestone IDs consistent across documents.
